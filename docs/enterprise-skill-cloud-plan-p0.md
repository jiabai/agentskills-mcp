# 执行计划 P0：基础能力与接口契约

**目标**：在不改变现有基线能力的前提下，补齐企业私有云技能服务的核心可用能力与对外接口契约。  
**建议工期**：2 周（可并行拆分为 2 个小组流）

## 范围（按优先级）
- 企业认证与身份映射：LDAP/AD/SSO 接入与组织/团队结构映射
- RBAC 与可见性控制：企业/团队/个人三级可见性落地
- MCP/REST 接口契约：`skill://list`、`skill://{id}@{version}`、`execute_skill`、`skills/download`
- 审计日志：采集、查询与导出（CSV/JSON）
- 技能版本：自动递增策略与校验

## 现有实现基线（可复用）
- 已支持 ZIP 上传、frontmatter 解析、版本归档/回滚、下架与缓存失效标记
- 已支持版本差异对比与依赖安装指引
- 已具备 API Token 与 JWT 基础认证链路

## 关键任务
- 认证与组织模型
  - LDAP/AD/SSO 接入与用户/组映射
  - 企业/团队/个人可见性字段与访问控制策略
- 权限与审计
  - RBAC 角色模型、权限矩阵与审计采集点
  - 审计日志查询与导出接口
- 接口契约
  - MCP 资源接口与工具执行契约补齐并验收
  - `skills/download` 加密传输接口对齐
- 版本管理
  - 版本自动递增与冲突处理策略

## 交付物
- MCP/REST 接口对齐说明与示例请求
- RBAC 与可见性模型设计与落地
- 审计日志查询与导出能力
- 版本自动递增实现与测试用例

## 验收标准
- 企业/团队/个人可见性在 API 与 MCP 入口一致生效
- `skill://` 资源接口与 `execute_skill` 可用且有权限控制
- 审计日志可按用户/时间/操作查询并导出
- 版本自动递增规则可配置且不产生冲突

## 依赖
- 企业目录与 SSO 接入配置可用
- 组织/团队结构数据源确定

## 风险与应对
- 组织映射复杂：先行支持最小字段集合，后续扩展
- 接口契约变化：保留兼容层并逐步迁移
