# 过去24小时统计清零接口化方案

## 背景
Dashboard 展示的“过去 24h 调用次数 / 调用成功率”来自 tool 调用的小时桶聚合（`request_metrics`）。现需要一个管理员可触发的操作，将“最近 24 小时窗口内”的统计数据清零，以便运维/演示场景快速重置指标。

## 目标
- 提供管理员接口，清零当前用户的“过去 24h”统计窗口
- 前端在 Dashboard 顶部操作区提供入口，明确其影响范围为“过去 24h 窗口”

## 接口设计
新增接口：`POST /api/v1/dashboard/metrics/reset-24h`
- 认证：JWT，且仅允许 `is_superuser=true` 用户调用
- 作用范围：仅清理当前登录用户的 `request_metrics` 中落在最近 24 小时窗口内的桶记录
- 返回：`removed`、`window_hours`、`window_start`、`window_end`

## 规则与异常处理
- 非管理员调用返回 403
- 清理失败返回 500，并记录日志

## 前端入口设计
- 入口位置：Dashboard 顶部操作区
- 与“清理历史统计”并列或收敛为一个“统计维护”入口，避免误解
- 交互：二次确认提示“仅清零过去 24h 窗口，不影响更早的历史统计”，执行后刷新概览

## 测试与验收
- 非管理员访问返回 403
- 管理员调用后，最近 24 小时桶被删除，窗口外桶不受影响
